    • Introduction


    In this blog I'm gonna tell you which is better architecture between MVP and MVVM. Before this I want to talk about why we need any architecture


    Need of Architecture


    1. Helps developers to change only business logic without affecting the entire system.
    2. Lower the maintainence cost.
    3. Developers can focus on business logic as an ideal architecture seperates the code.
    4. This makes easy for developers to test their code.


    What is MVP?


    MVP stands for Model-View-Presenter . Basically it is a software design pattern or you can say an architectural pattern which helps you to organize your code in three different parts which eases automated unit testing:
    Diagram of MVP architecture :

    Model

    1. It controls how data is created , stored in database.
    2. It is responsible for functionalities such as fetching of data from an API.
    3. Model can directly communicate with Presenter but not with View.

    View

    1. This component contains the visual part of an application and does not contain any business logic.
    2. In Android, View can be considered as Activities , Fragments or view.
    3. View is in one to one relationship with Presenter.

    Presenter

    1. The Presenter is in between Model and View.
    2. This component contains all the business logic of application.
    3. It receives data from model and shows it in View.


    Pros of MVP


    1. Testing is easy in MVP as business logic is seperated from UI.
    2. Low maintainence cost.
    3. The code written in View such as Activity or Fragment is comparatively less as all business logic is moved to Presenter.


    Cons of MVP


    1. Large number of interfaces.
    2. The code written in MVP is excessive


    What is MVVM


    MVVM stands for Model-View-ViewModel.

    Diagram of MVVM architecture :

    Model

    1. The Model layer of MVVM is similar to Model layer in MVP.
    2. It also contains data of application.
    3. Model layer can't communicate directly with View.
    4. It exposes data to View via ViewModels.

    View

    1. This layer contains the visual part of an application and does not contain any business logic.
    2. This layer observes the ViewModel.
    3. This layer also transmits data to ViewModel.

    ViewModel

    1. Viewmodel acts as a link between Model and View.
    2. Viewmodel retreives data from Model layer and sends to View, It doesn't care how the data is being used.
    3. It holds the actual reference to data for View.


    Pros of MVVM


    1. Less number of interfaces and classes as compared to MVP.
    2. The code written in MVVM is simplified.
    3. It allows the developers to change the code efficiently.


    Cons of MVVM


    1. In MVVM model , writing a test is comparatively difficult than MVP.
    2. In MVVM some code has to be written in XML so it increases the complexity in debugging.


    Diffrences between MVP and MVVM


    1. In MVVM , ViewModel replaces the Presenter in middle layer.
    2. In MVVM, View and ViewModel is in one to many relationship whereas in MVP ,View and Presenter is in one to one relationship.
    3. In MVP, number of classes is more as compared to MVVM.
    4. In MVVM, ViewModel holds reference to data whereas in MVP, Presener holds reference to View.
    5. In both MVVM and MVP , View is the entry point of application.
